# 💰 Butler 变现方案 v2.0

> 2026-06-12 改版。对齐用：先看本文档拍板，再落地代码。
> **v2.0 核心转向（合规重构）**：废弃 v1.1 的「按 token 成本 ×1.3 钱包扣费」——该形态把
> 计费单位直接绑在上游 token 上、公开加价率、卖点写成「充钱用 Claude/GPT」，与三家
> （Anthropic/OpenAI/Google）商业条款里 **「不得转售 API 访问本身」** 的红线高度模式匹配。
> v2.0 改为行业验证过的 **「产品积分/次数」** 模式（Cursor / Perplexity / Notion AI 同款）：
> 用户买的是 Butler 功能的使用次数，模型是实现细节 → 条款上是 Customer Product，不是 reseller。
>
> 沿用 v1.1 仍有效的部分：Flash 免费层获客、5h 滑动窗口、真实成本计量（转为内部核算）、
> 多供应商网关技术规格。v1.1 原文见 git 历史（commit 含「变现方案文档 v1.1」）。

## 0. 一句话商业模式

> **用「便宜到约等于免费」的 DeepSeek Flash 做获客与留存；用「Pro 专属功能 + 每月高级积分配额」做付费墙；超额卖一次性积分加油包；桌面版 / Obsidian 插件的重度用户走 BYOK（自带 key，调用零成本，功能仍按订阅档解锁）。**

```
                  ┌──────────────────────────────────────────────┐
  获客/留存  ──▶  │ Flash 免费层（5h 窗口分批，几乎用不完）          │  ← 日活、口碑
                  └──────────────────────────────────────────────┘
                                   │ 想要更强的功能 / 更聪明的回答
                                   ▼
                  ┌──────────────────────────────────────────────┐
  变现       ──▶  │ ① 订阅 Pro/Max：功能墙 + 每月积分配额（主线）    │  ← 主利润+现金流
                  │ ② 积分加油包：一次性次数包（副线，超额承接）      │
                  │ ③ BYOK：自带 key 免积分，功能仍需订阅（桌面王牌） │
                  └──────────────────────────────────────────────┘
```

**与 v1.1 的本质区别**（一层皮，但正好是合规边界）：

| | v1.1（废弃） | v2.0 |
|---|---|---|
| 计费单位 | ¥/token × 1.3，加价率公开 | 积分（一次功能调用 = N 积分） |
| 卖点话术 | 「充钱用 Claude/GPT，比官方便宜 25%」 | 「Pro 每月 300 积分 + 专属功能」 |
| 模型角色 | 商品本身（按模型计价表收费） | 体验档位（标准/增强，可注明 powered by） |
| 用户心智 | 我在买 token | 我在买 Butler 的高级能力 |
| 内部成本核算 | usage.ts 记真实成本 | **照旧保留**，只用于定积分汇率、盯毛利 |

## 1. 成本基线（内部核算用，不再对用户暴露）

> ⚠️ 代表性单价（USD/百万 token，约 ¥7.2/USD）。多供应商网关（§5）自动同步真实定价。
> v2.0 起这张表 **只服务内部**：定积分汇率、监控毛利。绝不出现在定价页。

| 模型 | 输入 $/M | 输出 $/M | 折 ¥/千token(输出) | 相对 Flash |
|---|---|---|---|---|
| **DeepSeek V4 Flash**（默认/免费层） | ~0.14 | ~0.28 | ~¥0.002 | 1× |
| DeepSeek V4 思考 | ~0.28 | ~0.42 | ~¥0.003 | ~1.5× |
| **Gemini 2.5 Pro** | ~1.25 | ~10 | ~¥0.072 | ~36× |
| **GPT（4o/5 级）** | ~2.5 | ~10 | ~¥0.072 | ~36× |
| **Claude Sonnet** | ~3 | ~15 | ~¥0.108 | ~54× |
| **Claude Opus** | ~15 | ~75 | ~¥0.54 | ~270× |

关键结论不变：Flash 免费层 ≈ 零成本获客；高端模型天然是付费墙——只是付费墙的形态从「token 钱包」换成「积分配额 + 功能墙」。

## 2. 免费层 spec（沿用 v1.1，已落地 P1+P2）

- Flash 免费额度按 **5h 滑动窗口** 分批发放（`lib/usage.ts`，每窗 ~¥0.6 内部成本当量，翻篇清零）。
- 窗口耗尽 → softwall（`QuotaWallModal`）：切回 Flash 继续免费 / 买加油包 / 开会员，带回满倒计时。
- **文案改造（v2.0 落地项）**：界面上把「¥0.40/¥0.60」改成次数/百分比表述（用量圆环 [088] 已是 %，方向对了）——金额单位只留在内部日志。
- 永远给「继续免费」出口，撞墙=流失。

## 3. 积分体系（v2.0 核心）

### 3.1 定义

- **积分 = Butler 高级功能的使用单位**。内部按「该操作的预估上游成本 × 目标毛利」折算，对用户只显示积分价。
- 汇率表（初版，内部按成本基线核算后微调）：

| 操作 | 积分 | 内部成本当量（参考） |
|---|---|---|
| 高级对话一轮（Gemini/GPT 档） | 1 | ~¥0.1-0.3 |
| 高级对话一轮（Claude Sonnet 档） | 2 | ~¥0.2-0.5 |
| 高级对话一轮（Claude Opus 档） | 5 | ~¥0.5-1.5 |
| AI 生成面板（generate-panel） | 2 | 一次 JSON 生成 |
| 深度调研·多小队并行（research） | 10 | plan + 3-4 squad 并行调用 |
| 多模态 OCR（Mistral，每 10 页） | 1 | $0.001/页 |
| Flash 对话 | 0（走免费窗口） | ~¥0.002/千token |

- **计价按「操作」不按 token**：用户发起前就能看到「本次操作消耗 N 积分」，确定性消费、算得清（解决 v1.1「按量算不清」焦虑）。超长对话内部用 token 估算分档（1/2/3 积分），对用户仍显示整数积分。
- 每条消息可展开「本条消耗 2 积分（增强模型）」——透明但不暴露 ¥/token。

### 3.2 为什么合规

1. 计费单位是 **产品功能使用次数**，与上游 token 解耦（内部换算是商业机密，不是商品标价）。
2. 不承诺「买到 Claude token」、不公开加价率、不做「比官方便宜」对比——没有任何转售表征。
3. 模型名只作为「体验档位说明」出现（如「增强档 · powered by Claude」），遵守各家品牌使用规范。
4. 平台 key 永远在服务端；终端用户受你的服务条款约束，你对 key 下的全部流量负责（→ §7 滥用防控）。

## 4. 定价（在现有 Free/Pro/Max 上重构）

| | Free | **Pro ¥39/月** | **Max ¥99/月** |
|---|---|---|---|
| Flash 对话 | ✅ 5h 窗口免费 | ✅ 同左 | ✅ 同左 |
| 高级模型档位 | 每月送 30 积分尝鲜 | ✅ 全档位 | ✅ 全档位 |
| **每月积分** | 30（+注册礼包一次性 50） | **300** | **1000** |
| 并行多小队调研 [075] | ❌ | ✅ | ✅ |
| AI 面板应用引擎 [073] | 3 个面板 | ✅ 无限 | ✅ 无限 |
| 历史记录 | 30 天 | 无限 | 无限 |
| 抢先体验新功能 | ❌ | ❌ | ✅ |
| 超额后 | 切 Flash / 买加油包 | 同左 | 同左 |
| **BYOK 自带 key** | ✅ 可填（功能仍按本档解锁） | ✅ 调用不耗积分 | ✅ 同左 |

- **积分加油包（一次性购买）**：¥10/100 分 · ¥30/350 分 · ¥50/650 分。12 个月有效，先进先出。
- 年付 8 折沿用；可加「年付送 100 积分」双重锚。
- 积分月配额不滚存（用不完清零，与行业一致）；加油包积分滚存。
- **付费理由从「模型更便宜」换成「功能更强」**：把最唬人的功能（并行调研、面板应用无限）锁 Pro，制造「为这个值得开会员」的哇时刻。

### 4.1 BYOK（自带 key）——桌面版 / Obsidian 插件的王牌

- 用户在偏好设置填自己的 ANTHROPIC/OPENAI/GEMINI key → 该供应商的调用 **不耗积分、不经平台 key**。
- **仅桌面版 / Obsidian 插件开放，网页版不开**（已拍板）：网页版是获客漏斗顶端，BYOK 的重度技术用户正好引导去桌面版；同时规避网页 localStorage 存 key 的 XSS 风险。
- **key 只存本地**（Tauri 安全存储 / Obsidian 插件设置），绝不上传服务器——这是本地化路线的天然优势，也彻底没有转售问题（用户和供应商直接结算）。
- 但 **高级功能仍按订阅档解锁**：BYOK 解决的是「模型成本」，订阅卖的是「Butler 的能力」。Obsidian 生态标准模式（Copilot / Text Generator 插件全是 BYOK + 付费功能），社区接受度已验证。
- 对你的价值：最重度的用户反而模型成本为零，订阅费全是毛利。

## 6. 记账与支付（Paddle 落地，已拍板）

> 上线前提：积分是真金白银，**必须服务端权威记账**（前端 localStorage 只做展示缓存）。
> 依赖 Phase 4（Clerk 鉴权 + Neon 数据库）先行。

- **支付通道 = Paddle（Merchant of Record）**（2026-06-12 拍板，初创默认配置）：
  - 大陆个人/无海外主体开不了 Stripe 标准账户；Stripe Atlas 注册美国 LLC ~$500 + 年度报税维护，对一人初创是纯负担。
  - Paddle 个人开发者可直接申请，作为 MoR 代收代缴全球 VAT/销售税、处理退款与合规——省掉整个税务后台。
  - 订阅（Pro/Max 月/年付）与积分加油包（one-time）Paddle 都原生支持，自带 customer portal。
- **billing-provider 薄抽象层**：`lib/billing-provider/`（createCheckout / webhook 验签 / 订阅状态查询接口统一）。以后有海外主体想切 Stripe = 换适配器，不是重写。
- **Webhook → 服务端积分账本**：`transaction.completed` 入账加油包、`subscription.activated/renewed` 发月配额、`subscription.canceled` 降档。账本表：`credits_ledger(userId, delta, reason, expiresAt)`。
- 每次高级操作：服务端先扣积分（原子操作，余额不足返回 softwall 信号）→ 再调上游。
- 若后续主打国内用户群，可在抽象层下加国内聚合支付（虎皮椒/Xorpay）作为第二适配器。

## 7. 风险与对策

| 风险 | 对策 |
|---|---|
| 条款合规（本版核心动机） | 积分计费 + 不暴露 token 价 + 品牌规范用名；商用注册各家 API；ToS/隐私政策上线前备好 |
| 平台 key 被滥用（你对全部流量负责） | 服务端鉴权（Clerk）+ 每用户限流 + 滥用监控；连接器 SSRF/env 注入收口（上线安全清单另列） |
| 免费层被刷 | 账号/设备/IP 限额 + 注册门槛（服务端记账后才有效） |
| 积分汇率失衡（毛利被打穿） | usage.ts 真实成本计量照旧跑 → 月度核对「积分收入 vs 上游账单」，汇率可调（条款注明） |
| 用户嫌积分抽象 | 操作前显示「本次 N 积分」+ 每条消息可展开明细；月配额「次数感」比 ¥/token 更易理解 |
| 免费太香没人付费 | 付费墙 = 功能（并行调研/面板无限）+ 模型档位双轮；Flash 永远免费保留存 |

## 9. 拍板记录（2026-06-12，按初创 SaaS 默认配置定）
